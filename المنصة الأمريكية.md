## 🧩 1) نبذة عن المشروع
منصة ويب **تعليمية** بالعربية تفحص وتحلّل **32 سهماً أمريكياً** وترصد فرص الانطلاق قبل حدوثها.
- **الهدف:** أداة فحص وتحليل تعليمية فقط — **لا توصيات ولا تنبؤات ولا وعود أرباح**.
- **صاحبها:** أحمد — مستخدم مبتدئ غير مبرمج، يفضّل شرحاً عربياً بسيطاً خطوة بخطوة وكلاماً قليلاً.

## 🧱 2) التقنية والبنية
- **الواجهة الخلفية:** Python + Flask
- **قاعدة البيانات:** PostgreSQL (عبر SQLAlchemy)
- **الاستضافة:** Railway (مشروع blissful-imagination) — **أي دمج في فرع `main` يُنشر تلقائياً على الموقع الحي خلال دقيقتين**.
- **الموقع الحي:** https://algomatix-production.up.railway.app (محمي بكلمة مرور APP_PASSWORD).
- **مستودع GitHub:** alasiri9/stock-analysis-platform

## 🔌 3) مصادر البيانات (من أين تجي الأرقام)
- **FMP** (Financial Modeling Prep) → بيانات مالية + سعر + تاريخ أسعار (لحساب ATR).
- **Finnhub** → سعر لحظي (مصدر تأكيد ثانٍ يرفع درجة الثقة).
- **SEC EDGAR** → معاملات المطّلعين (insiders) — بدون مفتاح.
- متغيرات البيئة على Railway: FMP_API_KEY, FINNHUB_API_KEY, DATABASE_URL, TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID, APP_PASSWORD.

## 🗄️ 4) جداول قاعدة البيانات (models.py)
- **StockCache** — (ticker, data_json, updated_at) نتائج الفحص المخزّنة.
- **Signal** — (ticker, signal_type, triggered_at, price_at_signal) الإشارات.
- **PricePoint** — (ticker, date, price) تاريخ الأسعار.
- **PortfolioHolding** — (ticker, shares, buy_price, user_id) المحفظة.
- **PortfolioSnapshot** — (date, total_cost, total_value) لقطات أداء المحفظة.
- **Watchlist** — (ticker, user_id, added_at, added_price) قائمة المتابعة.

## 📄 5) الصفحات (الروابط داخل الموقع)
- `/` — **الرئيسية**: الماسح + الفلاتر (Piotroski، النمو/Catalyst، السعر، القطاع، عدد المقاييس المجتمعة).
- `/stock/<رمز>` — **تقرير السهم**: سعر + مقاييس مالية + Piotroski + Catalyst + خطة ATR + مطّلعين + شارت.
- `/gems` — **الجواهر** • `/leaders` — **القادة** • `/prelaunch` — **ما قبل الانطلاق**.
- `/signals` — **الإشارات** • `/daily-report` — **التقرير اليومي**.
- `/radar` — **رادار المطّلعين** (EDGAR) • `/news` — **الأخبار** (Finnhub) • `/flow` — **التدفق**.
- `/performance` — **الأداء** • `/compare` — **المقارنة** • `/calculator` — **حاسبة**.
- `/watchlist` — **قائمة المتابعة** • `/portfolio` — **المحفظة** • `/login` `/logout`.
- `/learn` — **📚 تعلّم** (قاموس مصطلحات) • `/earnings` — **📆 رزنامة الأرباح** • `/alerts` — **🔔 تنبيهات سعرية**.

## 🧮 6) الحسابات (scoring.py + indicators.py)
- **Piotroski Score** (0–9) — محسوب يدوياً من FMP، بتفصيل كل نقطة.
- **Catalyst Score** (0–100) — سُمّي في الواجهة **«النمو»** (تمييزاً عن زخم السعر MACD). أيقونة القوة: 🟢 ≥80 / 🟡 40–79 / 🔴 <40.
- **خطة ATR تعليمية** — دخول / وقف / هدف (حساب تعليمي لا توصية). ATR يُحسب من أسعار FMP التاريخية لأن Finnhub المجاني يمنع الشموع (403).

## 🤖 7) الأتمتة (scheduler.py)
- **تحديث ليلي تلقائي 01:00 UTC** لكل الأسهم — غالباً لا حاجة لأزرار تحديث يدوية.
- **تقرير تلغرام صباحي** يُرسل تلقائياً.

## ⛔ 9) المبادئ الجوهرية (لا نحيد عنها)
1. **لا رقم بدون داتا حقيقية** — أي قيمة غير متوفرة تُعرض كـ «—» (لا صفر ولا رقم ملفّق).
2. **None ≠ 0** — نفرّق بوضوح بين بيانات غائبة وصفر حقيقي.
3. **Unit Guards إلزامية** — FMP يعيد نسباً مئوية (ROE=15.3 = 15.3%)، Finnhub بعضها كسور (0.153) — نعاير كل حقل مع تعليق يوضّح المصدر والوحدة.
4. **لا لغة توصية أو تنبؤ** — شاشة فحص تعليمية فقط.
5. كل النصوص عربية والاتجاه RTL.

## 🛠️ عناصر التطوير المقترحة (اذكرها لأحمد عندما يسأل «ماذا بقي من التطوير؟»)
> قائمة الأفكار الكبيرة المقترحة للمستقبل — راجعها وذكّر أحمد بها عند سؤاله عن التطوير القادم.

1. **📈 الاختبار التاريخي (Backtesting) — الأهم مصداقياً:** اختبار معايير المنصة (Piotroski/النمو/الإشارات) على بيانات الماضي (5–10 سنوات) لمعرفة نسبة نجاحها فعلاً تاريخياً («هذه الإشارة نجحت 68%») بدل «اجتهاد غير مُختبر». **العائق:** يحتاج **باقة FMP مدفوعة** (المجانية 250/يوم لا تكفي للبيانات التاريخية الضخمة) + جهد برمجي كبير (محرّك دقيق يتجنّب النظر للمستقبل وتحيّز الأسهم الناجية). **الوقت المناسب:** عند نمو دخل المشتركين → ترقية FMP → بناء المحرّك. سيكون قفزة كبيرة في قيمة المنصة.
2. **⬆️ ترقية باقة FMP** عند زيادة المشتركين الدافعين (تفتح الأسعار اللحظية للجميع + بيانات أكثر + الاختبار التاريخي).
3. **📩 تلغرام خاص لكل مشترك** لتنبيهاته السعرية (حالياً التنبيهات تصل لتلغرام المدير فقط).
4. **📊 تصنيف المحلّلين** في ميزان الإشارات (يحتاج بيانات محلّلين خارجية — غالباً محجوبة بالباقة المجانية).

## 🎨 «برواز الطبقات» — ستايل محفوظ لأحمد (يطلبه مستقبلاً لأي عنصر)
> أحمد أعجبه هذا البرواز على جدول مزاج الأسهم في الرئيسية، وطلب حفظه لإعادة استخدامه. عند طلبه «ضع برواز الطبقات على [عنصر]» طبّق هذا:

**البرواز الخارجي (طبقات مضيئة + توهّج نيون)** — عبر `box-shadow` على العنصر (مع `border-radius:14px`):
```css
box-shadow:
  0 0 0 3px #22c55e,                    /* طبقة داخلية: أخضر */
  0 0 0 6px #a7c0e8,                    /* طبقة أوسط: أزرق-فضّي (لون اسم Algomatix) */
  0 0 0 9px #7c6fe6,                    /* طبقة خارجية: أزرق-بنفسجي */
  0 0 26px 8px rgba(124, 111, 230, .45); /* توهّج نيون بنفسجي */
```
(يحتاج مسافة حول العنصر: `margin` + عرض أقل قليلاً مثل `width: calc(100% - 20px)` حتى تظهر الطبقات والتوهّج.)

**الشبكة/الخطوط الداخلية (نفس التدرّج لكن رفيعة)** — عبر `border-image` بتدرّج الألوان الثلاثة:
```css
border: 2.5px solid #7c6fe6;
border-image: linear-gradient(135deg, #22c55e, #a7c0e8, #7c6fe6) 1;
```

**الألوان الثلاثة المعتمدة:** أخضر `#22c55e` · أزرق-فضّي `#a7c0e8` · أزرق-بنفسجي `#7c6fe6`. (طُبّق أول مرة على `.mood-table` في الرئيسية — CSS v=20260720o.)
